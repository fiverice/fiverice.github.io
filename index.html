内部类

发表于 2016-05-15

将一个类的定义放在另一个类的定义内部，这就是内部类。

创建内部类

把类的定义置于外围类的里面。但通常更典型的应用场景为，外部类有一个方法，该方法返回一个指向内部类的引用，代码如下：

public class Parcel2 { 
    //创建内部类Destination
    class Destination {
        private String label;
        Destination(String whereTo) {
            label = whereTo;
        }
        String readLabel() {
            return label;
        }
    }
    //返回指向内部类引用的方法
    public Destination to(String s) {
        return new Destination(s);
    }
    public void ship(String dest) {
        Destination d = to(dest);
        System.out.println(d.readLabel());
    }
    public static void main(String[] args) {
        Parcel2 p = new Parcel2();
        p.ship("Beijing"); //通过外部类的方法创建内部类，并调用内部类对象的方法
        Parcel2.Destination d  = p.to("Shanghai"); //显式指明内部类的类型，创建内部类
        System.out.println(d.readLabel());
    }
}

如果想在外部类的非静态方法之外的任意位置创建内部类的对象，必须像在main()方法中一样，显式指明内部类的类型：OuterClassName.InnerClassName。

链接到外部类

内部类不仅是一种名字隐藏或组织代码的方式。当生成一个内部类对象时，此对象与生成它的外围对象就有了联系。它能访问外围对象的所有成员

嵌套类

如果不需要内部类对象与其外围类对象之间有联系，那么可以将内部类声明为static。这通常成为嵌套类，也叫static class。
请记住，普通内部类对象隐式地保存了一个引用，指向创建它的外围类对象，然而，当内部类声明为static时，情况发生了变化，这意味着：

  1. 要创建嵌套类的对象，并不需要其外围类的对象。
  2. 不能从嵌套类的对象中访问非静态的外围类对象。
  3. 普通内部类的字段与方法，只能放在类的外部层次上，所以普通内部类不能有static数据和static字段，也不能包含嵌套类，但是嵌套类可以包含所有这些东西。
